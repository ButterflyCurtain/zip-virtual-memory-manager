  The vmdirty file is an append-only binary journal that durably stores dirty
  pages spilled from the Diff Layer Tier 1 (in-memory) to Tier 2 (on-disk).
  It survives process crashes and provides the VMM with a complete, verifiable
  recovery basis.

  Design requirements addressed by this format:

    - Torn write detection: a crash mid-record must not produce an
      undetectable partial record.  Each record carries a length field and
      a trailing CRC so partial writes are identified precisely.

    - Per-record integrity: corruption anywhere in the file (not only at
      the tail) is detected by per-record CRC validation.

    - Commit boundaries: the file distinguishes pages that were explicitly
      checkpointed (durable) from pages that were in-flight at crash time.
      COMMIT MARKERs establish these boundaries.

    - Session identity: each record carries a generation ID so that spill
      files left by a prior mount session are unambiguously identified and
      not confused with the current session.

  SECTION 1 : Binary Format
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  All multi-byte integers are little-endian.
  All offsets are byte offsets from the start of the file.


  ┌──────────────────────────────────────────────────────────────────────────┐
  │  FILE HEADER  ( fixed 88 bytes, offset 0 )                               │
  │                                                                          │
  │  offset  size  field                  notes                              │
  │  ──────  ────  ─────────────────────  ─────────────────────────────────  │
  │   0      8     magic                  "VMDIRTY\0"                        │
  │                                       0x56 4D 44 49 52 54 59 00          │
  │   8      2     format_version         0x0001                             │
  │  10      2     flags                  see FLAG BITS below                │
  │  12     16     generation_id          see GENERATION ID below            │
  │  28      8     source_file_size       stat().st_size of archive.zip      │
  │  36      8     source_inode           stat().st_ino of archive.zip       │
  │  44     20     source_cd_hash         XXH3-128 of Central Directory      │
  │                                       block (16 bytes) + 4 bytes zero    │
  │                                       padding                            │
  │  64      8     created_at_ns          wall-clock ns; informational only  │
  │  72      4     page_size              bytes per page (e.g. 4096)         │
  │  76      4     reserved               0x00000000                         │
  │  80      4     header_crc32           CRC-32C of bytes [0 .. 79]         │
  │  84      4     padding                0x00000000                         │
  │                                                                          │
  │  Total: 88 bytes                                                         │
  └──────────────────────────────────────────────────────────────────────────┘

  FLAG BITS  ( flags field, 16 bits )

    bit 0   STRICT_FINGERPRINT    file was opened with --strict-fingerprint
    bit 1   SPILL_ONLY            mount was configured dirty_limit=0
    bit 2   RECOVERY_PENDING      diagnostic hint, not consulted by the
                                  recovery decision tree.  See "RECOVERY_
                                  PENDING protocol" below for the
                                  recommended set/clear discipline.
    bits 3–15  reserved; must be 0

  RECOVERY_PENDING protocol  ( recommended; informational )

    The recovery decision tree in Section 3 is complete without this
    bit: it classifies a found vmdirty solely by commit markers, valid
    record extents, and source fingerprint match.  RECOVERY_PENDING
    adds one diagnostic distinction — "the crash interrupted a commit"
    vs. "the crash interrupted spill/journal activity outside a
    commit" — useful for forensics and for triggering targeted orphan
    cleanup ( archive.new.zip, archive.zip.vmm/vmdirty.compact ) on a
    crash-during-commit recovery path.

    Recommended discipline:
      Set    : immediately before commit() makes any change that mutates
               archive.zip or writes archive.new.zip; persist with a
               single aligned 2-byte write at offset 10 of the file
               header followed by fdatasync().
      Clear  : after vmdirty deletion completes successfully; persist
               by the same single aligned 2-byte write followed by
               fdatasync().  ( Since vmdirty is about to be deleted,
               the clear is in practice elided — the absence of the
               file is the same signal. )

    Implementations MAY omit the bit's maintenance.  In that case the
    bit is permanently 0, recovery decisions are unaffected, and
    crash-during-commit is recognised by other state ( orphan
    archive.new.zip / vmdirty.compact, or the absence of a commit
    marker covering the highest sequence_num ).

    This is the only in-place mutation of the file header permitted
    by I-3.  The 2-byte flag field is naturally aligned and updated by
    a single write, so the mutation is atomic on all supported
    platforms.  No other header byte is mutated after creation.

  GENERATION ID  ( 16 bytes )

    A 128-bit value drawn entirely from a cryptographically secure random
    source at mount time.  It is structured as a UUID version 4 value:

        bytes[0..5]   : random
        bytes[6]      : 0x4?  ( version = 4, upper nibble fixed )
        bytes[7]      : 0x8? / 0x9? / 0xA? / 0xB?  ( variant bits )
        bytes[8..15]  : random

    The generation_id is not a sequence number and does not increment
    across sessions.  It identifies exactly one mount session.
    Collision probability across any two sessions is negligible:
    with 122 effective random bits, the probability of any collision
    across 10^12 mount operations is below 10^{-13}.


  ┌──────────────────────────────────────────────────────────────────────────┐
  │  DATA RECORD  ( variable length )                                        │
  │                                                                          │
  │  offset  size  field                  notes                              │
  │  ──────  ────  ─────────────────────  ─────────────────────────────────  │
  │   0      4     record_magic           0xD1A5EC0D                         │
  │   4     16     generation_id          must equal header.generation_id    │
  │  20      8     sequence_num           monotonically increasing, 1-based  │
  │  28      2     entry_name_len         byte length of entry_name (N)      │
  │  30      N     entry_name             UTF-8 ZIP entry path, no NUL       │
  │  30+N    8     page_index             0-based page index within entry    │
  │  38+N    4     data_len               equals header.page_size, except    │
  │                                       for the last page of an entry      │
  │                                       which may be smaller               │
  │  42+N    D     data                   raw uncompressed page bytes        │
  │                                       ( D = data_len )                   │
  │  42+N+D  4     record_crc32           CRC-32C of bytes [0 .. 41+N+D]    │
  │                                       covering record_magic through data  │
  │                                                                          │
  │  Total: 46 + N + D  bytes                                                │
  └──────────────────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  COMMIT MARKER  ( fixed 40 bytes )                                       │
  │                                                                          │
  │  offset  size  field                  notes                              │
  │  ──────  ────  ─────────────────────  ─────────────────────────────────  │
  │   0      4     marker_magic           0xC0FFEE42                         │
  │   4     16     generation_id          must equal header.generation_id    │
  │  20      8     commit_sequence        sequence_num of the last DATA      │
  │                                       RECORD included in this commit     │
  │  28      8     page_count             total dirty page count at this     │
  │                                       commit boundary                    │
  │  36      4     marker_crc32           CRC-32C of bytes [0 .. 35]         │
  │                                                                          │
  │  Total: 40 bytes                                                         │
  └──────────────────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  METADATA RECORD  ( variable length )                                    │
  │                                                                          │
  │  offset  size  field                  notes                              │
  │  ──────  ────  ─────────────────────  ─────────────────────────────────  │
  │   0      4     record_magic           0xD1A5EC1D                         │
  │   4     16     generation_id          must equal header.generation_id    │
  │  20      8     sequence_num           shared space with DATA RECORDs     │
  │  28      2     op_code                1=CREATE 2=REMOVE 3=RESIZE 4=RENAME│
  │  30      2     entry_name_len         byte length of entry_name (N)      │
  │  32      N     entry_name             UTF-8 ZIP entry path, no NUL       │
  │  32+N    P     op_payload             see OP PAYLOADS below              │
  │  32+N+P  4     record_crc32           CRC-32C of bytes [0 .. 31+N+P]     │
  │                                                                          │
  │  Total: 36 + N + P  bytes                                                │
  └──────────────────────────────────────────────────────────────────────────┘

  OP PAYLOADS

    CREATE  ( op_code = 1 )   payload: none ( P = 0 ).  logical_size = 0.
    REMOVE  ( op_code = 2 )   payload: none ( P = 0 ).  Tombstone.
    RESIZE  ( op_code = 3 )   payload: new_size : uint64  ( P = 8 ).
                              Emitted by truncate().  A shrink supersedes
                              journaled DATA RECORDs for pages beyond
                              new_size ( they become dead records ).
                              Implicit extension by write() emits no
                              RESIZE; see logical_size derivation in
                              Section 2.
    RENAME  ( op_code = 4 )   payload: new_name_len : uint16
                              + new_name : UTF-8, no NUL
                              ( P = 2 + new_name_len ).

  A METADATA RECORD participates in the same sequence_num ordering and
  COMMIT MARKER classification as DATA RECORDs.  Recovery replays records
  strictly in sequence order; an operation that follows a DATA RECORD for
  the same entry supersedes it in ordinary program order ( e.g. REMOVE
  invalidates earlier pages; CREATE after REMOVE begins a fresh entry ).

  CRC algorithm:

    CRC-32C (Castagnoli) is used for all CRC fields.
    Rationale: hardware-accelerated on x86 (SSE4.2 PCRC32) and ARM (CRC32
    extension); sub-microsecond on a 4 KB page on modern CPUs.  Better
    Hamming distance than CRC-32/ISO for short payloads.  Same polynomial
    used by iSCSI, SCTP, Btrfs, and ZFS for storage integrity.


  FILE STRUCTURE — complete view:

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  [ FILE HEADER           ]   88 bytes, always present                   │
  │  [ DATA RECORD  seq=1    ]   first spilled page                         │
  │  [ DATA RECORD  seq=2    ]                                               │
  │  [ DATA RECORD  seq=3    ]                                               │
  │  [ COMMIT MARKER         ]   logical checkpoint after seq=3             │
  │  [ DATA RECORD  seq=4    ]                                               │
  │  [ DATA RECORD  seq=5    ]                                               │
  │  [ DATA RECORD  seq=6    ]   ← process crashed here (partial write)     │
  │   ~~~~ EOF (file truncated mid-record) ~~~~                             │
  │                                                                          │
  │  On recovery:                                                            │
  │    seq 1–3  : fully valid, commit marker present → committed_pages      │
  │    seq 4–5  : valid records, past last commit marker → uncommitted       │
  │    seq 6    : partial write detected by CRC → discarded                 │
  └──────────────────────────────────────────────────────────────────────────┘

  SECTION 2 : Recovery Read Algorithm
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Executed once at open() when a vmdirty file is found.

  ─────────────────────────────────────────────────────────────────────────────
  function read_vmdirty(path) -> RecoveryResult:

    f = open(path, O_RDONLY)

    ── STEP 1: Validate header ──────────────────────────────────────────────
    hdr = f.read(88)
    if len(hdr) < 88:
        return RecoveryResult(status=HEADER_TRUNCATED)
    if hdr[0:8] != b"VMDIRTY\0":
        return RecoveryResult(status=HEADER_MAGIC_BAD)
    if crc32c(hdr[0:80]) != uint32_le(hdr[80:84]):
        return RecoveryResult(status=HEADER_CRC_BAD)

    gen_id        = bytes(hdr[12:28])           // 16-byte generation ID
    page_size     = uint32_le(hdr[72:76])
    format_ver    = uint16_le(hdr[8:10])
    if format_ver > SUPPORTED_VERSION:
        return RecoveryResult(status=VERSION_UNSUPPORTED)

    ── STEP 2: Walk records ─────────────────────────────────────────────────
    valid_records    = []
    last_commit_seq  = 0          # 0 = no commit marker seen
    truncation_point = None       # byte offset where truncation was detected

    pos = 88
    while True:
        magic_buf = f.read(4)
        if len(magic_buf) == 0:
            break                 # clean EOF

        if len(magic_buf) < 4:
            truncation_point = pos
            break

        magic = uint32_le(magic_buf)

        ── DATA RECORD ──────────────────────────────────────────────────────
        if magic == 0xD1A5EC0D:
            fixed = f.read(26)          // gen(16) + seq(8) + name_len(2)
            if len(fixed) < 26:
                truncation_point = pos
                break

            record_gen  = bytes(fixed[0:16])
            seq_num     = uint64_le(fixed[16:24])
            name_len    = uint16_le(fixed[24:26])

            if record_gen != gen_id:
                truncation_point = pos
                break

            name_and_rest = f.read(name_len + 8 + 4)
            if len(name_and_rest) < name_len + 12:
                truncation_point = pos
                break

            entry_name = name_and_rest[0:name_len].decode('utf-8', errors='replace')
            page_index = uint64_le(name_and_rest[name_len:name_len+8])
            data_len   = uint32_le(name_and_rest[name_len+8:name_len+12])

            if data_len > page_size:
                truncation_point = pos
                break

            data = f.read(data_len)
            if len(data) < data_len:
                truncation_point = pos
                break

            crc_buf = f.read(4)
            if len(crc_buf) < 4:
                truncation_point = pos
                break

            covered = (magic_buf + fixed + name_and_rest + data)
            if crc32c(covered) != uint32_le(crc_buf):
                truncation_point = pos
                break

            valid_records.append(DirtyPage(
                entry_name = entry_name,
                page_index = page_index,
                data       = data,
                sequence   = seq_num,
            ))
            pos = f.tell()

        ── METADATA RECORD ──────────────────────────────────────────────────
        elif magic == 0xD1A5EC1D:
            fixed = f.read(28)          // gen(16) + seq(8) + op(2) + name_len(2)
            if len(fixed) < 28:
                truncation_point = pos
                break

            record_gen  = bytes(fixed[0:16])
            seq_num     = uint64_le(fixed[16:24])
            op_code     = uint16_le(fixed[24:26])
            name_len    = uint16_le(fixed[26:28])

            if record_gen != gen_id or op_code not in (1, 2, 3, 4):
                truncation_point = pos
                break

            payload_len = { 1: 0, 2: 0, 3: 8 }.get(op_code)   // RENAME: read below
            name_buf = f.read(name_len)
            if len(name_buf) < name_len:
                truncation_point = pos
                break

            if op_code == 4:            // RENAME: peek new_name_len
                len_buf = f.read(2)
                if len(len_buf) < 2:
                    truncation_point = pos
                    break
                payload_len = 2 + uint16_le(len_buf)
                payload = len_buf + f.read(payload_len - 2)
            else:
                payload = f.read(payload_len)
            if len(payload) < payload_len:
                truncation_point = pos
                break

            crc_buf = f.read(4)
            covered = magic_buf + fixed + name_buf + payload
            if len(crc_buf) < 4 or crc32c(covered) != uint32_le(crc_buf):
                truncation_point = pos
                break

            valid_records.append(EntryOp(
                op         = op_code,
                entry_name = name_buf.decode('utf-8', errors='replace'),
                payload    = payload,
                sequence   = seq_num,
            ))
            pos = f.tell()

        ── COMMIT MARKER ────────────────────────────────────────────────────
        elif magic == 0xC0FFEE42:
            rest = f.read(36)           // gen(16) + commit_seq(8) + page_count(8) + crc(4)
            if len(rest) < 36:
                truncation_point = pos
                break

            marker_gen    = bytes(rest[0:16])
            commit_seq    = uint64_le(rest[16:24])
            covered       = magic_buf + rest[0:32]
            expected_crc  = uint32_le(rest[32:36])

            if marker_gen != gen_id or crc32c(covered) != expected_crc:
                truncation_point = pos
                break

            last_commit_seq = commit_seq
            pos = f.tell()

        ── UNKNOWN MAGIC ────────────────────────────────────────────────────
        else:
            truncation_point = pos
            break

    ── STEP 3: Classify result ──────────────────────────────────────────────
    pages = [r for r in valid_records if isinstance(r, DirtyPage)]
    ops   = [r for r in valid_records if isinstance(r, EntryOp)]

    committed_pages   = [r for r in pages if r.sequence <= last_commit_seq]
    uncommitted_pages = [r for r in pages if r.sequence >  last_commit_seq]
    committed_ops     = [r for r in ops   if r.sequence <= last_commit_seq]
    uncommitted_ops   = [r for r in ops   if r.sequence >  last_commit_seq]

    return RecoveryResult(
        status             = OK,
        generation_id      = gen_id,
        committed_pages    = committed_pages,
        uncommitted_pages  = uncommitted_pages,
        committed_ops      = committed_ops,
        uncommitted_ops    = uncommitted_ops,
        last_commit_seq    = last_commit_seq,
        truncation_point   = truncation_point,
        valid_through_seq  = valid_records[-1].sequence if valid_records else 0,
    )
  ─────────────────────────────────────────────────────────────────────────────

  logical_size derivation during replay:

    Replaying the selected records (pages and ops interleaved in strict
    sequence order) reconstructs each entry's logical_size:

      base          = uncompressed_size from the Central Directory
                      ( 0 after CREATE; undefined after REMOVE )
      RESIZE        sets logical_size = new_size
      DATA RECORD   sets logical_size = max( logical_size,
                                page_index × page_size + data_len )

    The max() rule recovers implicit extension by write() without a
    dedicated record: the DATA RECORD for an entry's final page carries
    data_len = logical_size − page_start.  Pages inside an extension gap
    that were never written are absent from the journal and read as
    zeros.

  SECTION 3 : Recovery Protocol — Caller API and Decision Tree
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  The VMM surfaces a RecoveryResult to the caller.  The VMM auto-resolves
  only the two unambiguous branches: a stale empty journal is silently
  discarded, and a clean commit boundary ( last_commit_seq > 0 with no
  uncommitted records ) is recovered via recover_committed().  For every
  branch that could lose data, the VMM does not auto-resolve; the
  data-safety decision remains with the caller.

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  RecoveryResult fields                                                   │
  │                                                                          │
  │  status             OK | HEADER_TRUNCATED | HEADER_MAGIC_BAD |          │
  │                     HEADER_CRC_BAD | VERSION_UNSUPPORTED                 │
  │  generation_id      identifies the mount session that wrote this file    │
  │  committed_pages    pages whose sequence ≤ last commit marker            │
  │  uncommitted_pages  pages whose sequence > last commit marker            │
  │  committed_ops      entry operations whose sequence ≤ last commit marker │
  │  uncommitted_ops    entry operations whose sequence > last commit marker │
  │  last_commit_seq    0 if no commit marker was ever written               │
  │  truncation_point   byte offset of detected truncation; None if clean   │
  │  valid_through_seq  highest sequence number that passed CRC validation   │
  └──────────────────────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  Caller resolution options                                               │
  │                                                                          │
  │  recover_committed()                                                     │
  │    Replay committed_ops into the entry table and restore                 │
  │    committed_pages into Tier 1, interleaved in sequence order.           │
  │    Discard uncommitted_pages and uncommitted_ops.                        │
  │    Safe for any automated recovery.                                      │
  │                                                                          │
  │  recover_all()                                                           │
  │    Replay all ops and restore all pages, committed and uncommitted,      │
  │    interleaved in sequence order.                                        │
  │    Caller accepts that uncommitted pages may represent partial state.    │
  │    Suitable for single-writer applications with no logical transaction   │
  │    boundary.                                                             │
  │                                                                          │
  │  discard()                                                               │
  │    Delete vmdirty.  All dirty pages are lost.                           │
  │    Mount proceeds clean.                                                 │
  │    Suitable for read-only use cases or when caller can regenerate.       │
  │                                                                          │
  │  abort()                                                                 │
  │    Leave vmdirty on disk untouched.  Do not mount.                     │
  │    Suitable for manual inspection before a decision is made.             │
  └──────────────────────────────────────────────────────────────────────────┘

  Decision tree presented to caller at open():

  open( archive.zip )
       │
       ├─ vmdirty absent → mount CLEAN
       │
       └─ vmdirty present
               │
               ├─ fingerprint check: source ZIP changed while dirty pages exist?
               │       YES → CONFLICT state
               │               offer: discard() | abort()
               │       NO  → run read_vmdirty()
               │
               └─ RecoveryResult.status
                       │
                       ├─ HEADER_TRUNCATED | HEADER_MAGIC_BAD | HEADER_CRC_BAD
                       │       file unreadable
                       │       offer: discard() | abort()
                       │
                       ├─ VERSION_UNSUPPORTED
                       │       upgrade required
                       │       offer: abort()
                       │
                       └─ OK
                               │
                               ├─ last_commit_seq == 0
                               │    AND committed_pages empty
                               │    AND uncommitted_pages empty
                               │       → stale empty file → silently discard()
                               │
                               ├─ last_commit_seq == 0
                               │    AND uncommitted_pages non-empty
                               │       → no checkpoint was written
                               │       → offer: recover_all() | discard() | abort()
                               │
                               ├─ last_commit_seq > 0
                               │    AND uncommitted_pages empty
                               │       → clean crash with commit boundary
                               │       → auto: recover_committed()  ← safe default
                               │
                               └─ last_commit_seq > 0
                                   AND uncommitted_pages non-empty
                                       → in-flight work exists past last checkpoint
                                       → offer: recover_committed()
                                               | recover_all()
                                               | discard()
                                               | abort()

  Wherever the tree tests committed_pages / uncommitted_pages, the
  corresponding ops lists are included in the same test: a journal
  containing only METADATA RECORDs ( e.g. a session that only renamed
  or removed entries ) is still a recoverable dirty session.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  SECTION 4 : Write Path
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  4.1  When a DATA RECORD is written

  A DATA RECORD is appended exactly when a dirty page is evicted from
  Tier 1 to Tier 2.

  Trigger condition:
      dirty_current + incoming_page_size > dirty_limit

  Write procedure:

    a. Select victim pages per spill policy (see Section 5).
    b. For each victim page:
         i.   Build DATA RECORD in a write buffer.
         ii.  Compute CRC-32C over the buffer (excluding the crc field).
         iii. Append CRC to buffer.
         iv.  Write buffer to vmdirty with a single write() call.
              ( Single call minimizes the torn-write window. )
    c. Remove victim page from Tier 1.  Decrease dirty_current.
    d. Update the in-memory Tier 2 index:
         (entry_name, page_index) → byte_offset_in_vmdirty

  fsync policy:

    Default (--sync-spill):
        vmdirty is opened with O_DSYNC.
        Each write is durable before returning.

    Fast mode (--lazy-spill):
        Records are buffered in OS page cache.
        fdatasync() is called only when a COMMIT MARKER is written.
        Crash between a spill and the next commit marker loses in-flight
        pages.  Acceptable when the application will retry lost writes.

  Write hit on a page already in Tier 1:
      Update data in-place.
      Re-stamp as newest (move to back of FIFO deque).
      dirty_current is unchanged.

  Write hit on a page already in Tier 2:
      Append a new DATA RECORD to vmdirty (higher sequence_num).
      The new record supersedes the prior one for recovery purposes.
      The page is not moved back to Tier 1.


  4.2  When a COMMIT MARKER is written

  A COMMIT MARKER is written when the application calls flush().

  flush() procedure (STRICT mode, default):

    a. Spill all Tier 1 pages that exceed dirty_limit.
    b. Spill all remaining Tier 1 pages.
    c. Build COMMIT MARKER with commit_sequence = highest sequence_num.
    d. Write to vmdirty.
    e. fdatasync().
    f. Update in-memory last_commit_seq.

  After flush() returns (STRICT), a crash is recoverable via
  recover_committed(), which restores the full dirty state as of this call.

  flush() (LAZY mode, --lazy-flush):

    Steps a and b are omitted.  The COMMIT MARKER covers only pages
    already in Tier 2 at the time of the call.  Tier 1 pages not yet
    spilled are not covered.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  SECTION 5 : Spill Policy
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  5.1  FIFO Spill Policy (default)

  When dirty_limit is exceeded, the oldest-written dirty pages are spilled
  first.  Rationale: recently-written pages exhibit higher temporal locality
  of re-mutation; preserving them in Tier 1 reduces write amplification in
  common workloads.  The policy is also the simplest correct implementation,
  with no per-page metadata beyond write time.

  Tier 1 data structure:

    Doubly-linked list (insertion order) + hash map for O(1) lookup.

    ┌──────────────────────────────────────────────────────────────────────┐
    │  insertion_order (deque):                                            │
    │    OLDEST ──→ [ file_a:pg0 ][ file_b:pg12 ][ file_c:pg0 ] ──→ NEWEST│
    │                                                                      │
    │  index (HashMap):                                                    │
    │    (file_a, 0)  → &node                                              │
    │    (file_b, 12) → &node                                              │
    │    (file_c, 0)  → &node                                              │
    └──────────────────────────────────────────────────────────────────────┘

  Spill selection:

      while dirty_current + incoming > dirty_limit:
          victim = insertion_order.pop_front()    // oldest
          write DATA RECORD to vmdirty
          remove from index
          dirty_current -= page_size

  Write hit on a Tier 1 page:

      Update data in-place.
      Move node to back of insertion_order (re-stamp as newest).
      dirty_current is unchanged.

  Write hit on a Tier 2 page:

      Append new DATA RECORD (higher sequence_num) to vmdirty.
      New record supersedes old one for recovery.
      Page remains in Tier 2; it is not moved back to Tier 1.

  Known write amplification case:

    1. Write page A  → A is the oldest dirty page
    2. Write 63 other pages → fills Tier 1 near dirty_limit
    3. Write one more page  → pressure → FIFO spills A to Tier 2
    4. Write page A again   → appends another DATA RECORD to vmdirty

    A temperature-aware policy would detect A as hot and spare it from
    spill.  This is a correctness-neutral trade-off; the FIFO policy
    accepts higher write amplification in exchange for implementation
    simplicity.


  5.2  Temperature-Aware Spill Policies (optional, mount-time selection)

  The vmdirty format is policy-agnostic.  Only the victim selection
  algorithm in Tier 1 changes.  No format migration is required.

  LRU policy:

    Replace the insertion_order deque with an LRU list maintained by
    both write time and read time.  On each page access (read or write),
    move the node to the back (most-recently-used).  Spill selects from
    the front (least-recently-used dirty page).

  Clock-hand policy (lower overhead than strict LRU):

    Each dirty page carries a reference bit (ref_bit = 0 initially).
    A logical clock hand sweeps Tier 1 during spill selection:
        if ref_bit == 1: clear ref_bit, advance hand
        if ref_bit == 0: this page is the spill victim
    On any access to the page: ref_bit = 1.

  Mount options:

    mount(path, spill_policy=FIFO)    // default
    mount(path, spill_policy=LRU)
    mount(path, spill_policy=CLOCK)


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  SECTION 6 : Generation ID and Session Provenance
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  A vmdirty file left on disk by a prior mount session must be
  distinguishable from one written by the current session.  This matters
  especially on network-mounted filesystems where multiple hosts may share
  the same archive path.

  Provenance check on open():

    The VMM does NOT compare generation IDs for equality between sessions.
    A vmdirty from a prior session always has a different generation ID;
    this is expected.  The relevant check is source fingerprint:

        vmdirty.header.source_file_size == stat(archive.zip).st_size
        AND
        vmdirty.header.source_inode     == stat(archive.zip).st_ino
        AND
        vmdirty.header.source_cd_hash   == current cd_hash

    Fingerprint matches → vmdirty is from a prior mount of THIS archive.
        The generation_id identifies which prior session.  Proceed with
        the recovery protocol.

    Fingerprint mismatches → vmdirty is stale (source changed).
        CONFLICT state.  Caller must resolve.

  Recovery and session transition:

    On a fresh mount after recovery:
        old vmdirty → rename to vmdirty.bak.{generation_id hex}
        new vmdirty → created with a new generation_id
        vmdirty.bak.* → retained for forensics; not used by the VMM

    Each DATA RECORD carries the generation_id from the file header.
    A generation_id mismatch within a record terminates record walking
    at that point (detected as corruption, not as a valid prior session).


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  SECTION 7 : Component Interfaces
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  VmdirtyWriter

    createFile(path, fingerprint, generation_id, page_size)
        Writes FILE HEADER with CRC.  Opens file with O_DSYNC (default)
        or buffered (--lazy-spill).

    appendRecord(entry_name, page_index, data) → sequence_num
        Builds DATA RECORD, computes CRC-32C, writes atomically.

    appendMetadata(op_code, entry_name, payload) → sequence_num
        Builds METADATA RECORD (CREATE / REMOVE / RESIZE / RENAME),
        computes CRC-32C, writes atomically.  Shares the sequence_num
        counter with appendRecord().

    appendCommitMarker(commit_sequence)
        Writes COMMIT MARKER, calls fdatasync().

    close()


  VmdirtyReader

    readAll(path) → RecoveryResult
        Validates header.  Walks records.  Classifies committed and
        uncommitted pages.  Returns RecoveryResult as described in
        Section 3.


  Tier1DirtyStore

    write(entry_name, page_index, data)
        Checks dirty_limit.  Triggers spill via VmdirtyWriter if exceeded.
        In synchronous spill mode: spill executes inline before returning.
        In asynchronous spill mode: page is enqueued; caller returns
        immediately unless backpressure threshold is reached
        (see Diff Layer Pressure specification).
        Updates FIFO deque and index.

    read(entry_name, page_index) → Optional<bytes>
        Checks Tier 1 first; falls back to VmdirtyIndex for Tier 2.

    spill(n_pages)
        FIFO victim selection.  Delegates writes to VmdirtyWriter.


  VmdirtyIndex  ( in-memory only )

    Populated at mount from an existing vmdirty when present.
    Maps (entry_name, page_index) → byte_offset_in_vmdirty.
    Used by the Tier 2 read path.
    Rebuilt entirely from vmdirty on recovery; never persisted separately.


  VmmMount

    open(path, options)
        Acquires advisory lock on archive.zip.vmm/vmlock.
        Runs fingerprint validation against archive.zip.
        Loads or builds vmidx (per the configured index strategy) so
            the entry table is available before any recovery replay.
            Recovery replays METADATA RECORDs ( CREATE / REMOVE / RESIZE
            / RENAME ) against the entry overlay, which is layered on
            top of vmidx's entry table; vmidx must exist first.
        Runs VmdirtyReader if vmdirty is present.
        Routes to recovery decision tree.
        Exposes RecoveryResult to caller.
        Removes orphan archive.zip.vmm/vmdirty.compact and archive.new.zip
        if present with no active mount.

    RecoveryHandler interface
        onRecovery(RecoveryResult) → RecoveryDecision
            { RECOVER_COMMITTED | RECOVER_ALL | DISCARD | ABORT }

    applyRecovery(decision, result)
        Loads committed_pages (and optionally uncommitted_pages) into Tier 1.
        Renames old vmdirty to vmdirty.bak.{generation_id hex}.
        Creates new vmdirty for the current session.

    flush()
        STRICT: spill all Tier 1 pages → appendCommitMarker() → fdatasync()
        LAZY:   appendCommitMarker() for currently-spilled pages only

    commit()
        Calls flush() (STRICT).
        Selects INCREMENTAL or FULL path based on bloat_ratio and gc_threshold.
        INCREMENTAL: appends modified entry data + new CD + EOCD to archive.zip.
        FULL: recompress → write archive.new.zip → rename().
        On success: deletes vmdirty; updates vmidx for modified entries.
        On failure before archive modification: vmdirty remains intact;
        recovery path is intact.

    compact()
        Permitted only from CLEAN state (no dirty pages).
        Performs FULL (compaction) commit with an empty Diff Layer.
        Eliminates accumulated dead data in archive.zip.
        Does not touch vmdirty (none is present in CLEAN state).

    compactJournal()
        Triggered internally when dead_record_count > live_record_count.
        Executes the compaction procedure defined in Section 10.
        Not directly callable by the application; it is an internal
        maintenance operation of Tier1DirtyStore.

  SECTION 8 : Invariants and Safety Properties
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  The following invariants must hold at all times.  Any violation is a VMM
  bug, not a recoverable runtime condition.

  I-1  Source ZIP is read-only until commit().
         archive.zip is never opened with write intent until commit() begins.

  I-2  Dirty pages are never silently discarded.
         A page in Tier 1 or Tier 2 is removed only by:
           (a) commit()             — page written to new ZIP
           (b) discard()           — explicit caller decision
           (c) recover_committed() / recover_all() — recovery decision
         It is never removed by LRU eviction, OOM killer, or any implicit path.

  I-3  vmdirty grows monotonically within a session, except during compaction.
         Records are only appended, never overwritten in place.
         Exception 1: the header flags field may be updated in place for the
         RECOVERY_PENDING bit (single aligned write; safe on all platforms).
         Exception 2: compaction (Section 10) replaces vmdirty atomically
         via rename() of archive.zip.vmm/vmdirty.compact, producing a smaller
         file with no dead records.  After rename(), monotonic growth resumes.

  I-4  A COMMIT MARKER sequence number is always ≥ the previous commit
         sequence.  Commit markers are never written out of order.

  I-5  sequence_num is strictly monotonically increasing per generation.
         No two records (DATA or METADATA) in the same vmdirty file share
         a sequence_num.

  I-6  The in-memory VmdirtyIndex is always consistent with vmdirty on disk.
         If a record was appended to disk, it is present in the index or
         superseded by a newer record for the same page.

  I-7  After flush() returns (STRICT mode), all dirty pages are durable.
         A crash after flush() is fully recoverable via recover_committed().

  I-8  At most one process holds LOCK_EX on archive.zip.vmm/vmlock at any time.
         This is enforced by the kernel; no additional in-process coordination
         is required beyond acquiring the lock at open() and releasing at
         close().


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  SECTION 9 : Sidecar File Naming and Co-location Rules
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  archive.zip                source ZIP (read-only until commit)
  archive.zip.vmm/           sidecar directory (created at first open)
    vmidx                    seek index (rebuilt if absent or stale)
    vmlock                   advisory lock coordination file
    vmdirty                  spill journal (current session)
    vmdirty.bak.{gen_id}     recovered journal (forensics retention)

  If archive.zip is moved, the sidecar directory must move with it.
  Never delete the sidecar directory while a mount is active (see main
  architecture document, SIDECAR FILES, deletion warning).

  A ZIP without its vmidx is still a valid ZIP; first-open rebuilds the index.
  A ZIP without its vmlock is handled by creating the file at next open().

  A ZIP without its vmdirty means dirty pages from a prior session are lost.
  The source ZIP itself is unaffected.

  On commit() success:
    vmdirty is deleted.
    vmidx is updated for modified entries only.
    vmdirty.bak.* files are retained for a configurable period
    (default: 24 hours, or deleted at next clean mount).

  On close() without a preceding commit() (DIRTY close):
    vmdirty is NOT deleted.  It remains on disk with whatever records
    were written and whatever last_commit_seq was established by the
    most recent flush(), if any.  The next open() finds vmdirty present
    and runs the recovery protocol: a clean-flush DIRTY close presents
    last_commit_seq > 0 with uncommitted_pages empty and is auto-resolved
    via recover_committed(); a DIRTY close without flush() presents
    last_commit_seq = 0 with uncommitted_pages non-empty and is offered
    to the caller per Section 3 of this specification.

  On close() from CLEAN state:
    No vmdirty exists.  The mount releases its advisory lock and returns.
    No sidecar mutation.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  SECTION 10 : Journal Compaction
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  The vmdirty file grows monotonically within a session.  When the same
  page is written multiple times and spills to Tier 2 on each occasion,
  a new DATA RECORD is appended for each spill while the prior records for
  the same (entry_name, page_index) remain on disk as unreferenced data.
  Compaction eliminates these dead records, bounding vmdirty size to
  one DATA RECORD per unique live dirty page.

  Terminology:

    live record  :  the highest-sequence DATA RECORD for a given
                    (entry_name, page_index) pair; the record currently
                    indexed by VmdirtyIndex.
    dead record  :  any DATA RECORD superseded by a later record for the
                    same (entry_name, page_index) pair, or invalidated by
                    a later METADATA RECORD ( REMOVE of the entry; RESIZE
                    shrink past the page ).  A METADATA RECORD is dead
                    when a later operation on the same entry supersedes
                    its effect ( e.g. CREATE followed by REMOVE ).


  Compaction trigger:

    Compaction is evaluated after each spill operation.  The condition is:

        dead_record_count  >  live_record_count

    This means more than half the records on disk are dead.
    dead_record_count is tracked in-memory (incremented on each write hit
    to a Tier 2 page) and requires no disk scan to evaluate.


  Compaction procedure:

    1. Build a write buffer containing:

         a.  FILE HEADER with a new generation_id drawn from a
             cryptographically secure random source.

         b.  One METADATA RECORD per entry whose lifecycle state differs
             from the source archive ( created / removed / resized /
             renamed ), reflecting the FINAL state only — intermediate
             operation history is not preserved.  Emitted before that
             entry's DATA RECORDs so that replay order is valid.

         b2. One DATA RECORD per entry in VmdirtyIndex, using the live page
             data.  Sequence numbers are assigned 1, 2, 3, … in arbitrary
             entry order (the new generation_id makes sequence numbering
             independent of the prior session).

         c.  A COMMIT MARKER with commit_sequence = highest assigned
             sequence number, if and only if all live records were
             previously committed in the prior session
             ( i.e. original_sequence ≤ last_commit_seq for every live page ).
             If any live record was uncommitted, no COMMIT MARKER is written.

    2. Write the buffer to  archive.zip.vmm/vmdirty.compact.

    3. fdatasync()  on  archive.zip.vmm/vmdirty.compact.

    4. rename( archive.zip.vmm/vmdirty.compact → archive.zip.vmm/vmdirty ).

    5. Rebuild VmdirtyIndex by reading the compacted file from the start
       ( offsets have changed ).

    6. Set last_commit_seq to the new commit_sequence if a COMMIT MARKER
       was written in step 1c; otherwise set last_commit_seq = 0.

    7. Reset dead_record_count = 0.


  Crash safety:

    Crash before step 4 :  archive.zip.vmm/vmdirty is unchanged.
                           archive.zip.vmm/vmdirty.compact is an orphan;
                           it is removed silently at next open().
    Crash after step 4  :  the compacted file is the recovery basis.
                           Its format is identical to an ordinary vmdirty
                           and is processed by the standard recovery protocol
                           in Section 3.  No special recovery case is required.


  Commit marker semantics after compaction:

    Pages that were committed before compaction remain committed after it.
    The COMMIT MARKER in the compacted file re-establishes that boundary.
    Pages that were uncommitted before compaction remain uncommitted after it;
    no COMMIT MARKER covers them in the compacted file.

    Consequence for flush() guarantee (I-7):
      If flush() (STRICT) completed before compaction, all dirty pages were
      committed.  The compacted file carries a COMMIT MARKER covering all
      live records.  A crash after compaction is fully recoverable via
      recover_committed().  The guarantee is not weakened.


  Invariants preserved after compaction:

    I-2  dirty_current is unchanged.  Compaction operates on Tier 2 only;
         Tier 1 content is not moved or affected.

    I-3  VmdirtyIndex is rebuilt in step 5 to be consistent with the
         compacted file.

    I-5  sequence_num is strictly monotonically increasing within the
         new file.  Renumbering from 1 is valid because the new file
         carries a new generation_id; no prior record carries the new
         generation_id.

    I-7  flush() durability guarantee is preserved as described above.
